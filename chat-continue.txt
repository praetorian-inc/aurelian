  list-all already covers the same ground with more detail.

⏺ This one is worth migrating. Here's the breakdown:

  Nebula's role-assignments module today

  - Enumerates all role assignments at subscription + resource group +
  management group scopes
  - Resolves role definition IDs → display names (e.g., "Owner", "Contributor")
  - Does NOT resolve principal IDs (hardcoded to "Unknown" — no Graph API calls)
  - No security analysis — pure enumeration, no risk flagging
  - Output: raw JSON + Markdown table

  Why migrate it

  - Aurelian has no equivalent — the only role-related code is a narrow VM
  identity check in configscan/vm_identity.go
  - Role assignments are foundational data for Azure security — you need them to
   detect:
    - Overprivileged principals (Owner/Contributor at subscription scope)
    - Standing access vs JIT
    - Stale assignments
    - Service principals with excessive permissions
  - It fills a gap — list-all enumerates resources, but not IAM bindings

  What to improve during migration

  1. Resolve principal IDs via Microsoft Graph (users, groups, service
  principals) — reuse the graphresolver component you already built for
  conditional access policies
  2. Emit AurelianRisk findings for high-risk patterns (e.g., Owner at
  subscription scope, classic admin roles, external principals)
  3. Use ARG instead of ARM pagination — authorizationresources table in ARG can
   query role assignments more efficiently
  4. Add principal type detection — Nebula leaves it as "Unknown"

  TL;DR

  ┌──────────────────────┬─────────────────────────┬─────────────────────────┐
  │                      │ Nebula role-assignments │   Aurelian (proposed)   │
  ├──────────────────────┼─────────────────────────┼─────────────────────────┤
  │ Scope                │ Sub + RG + MgmtGroup    │ Same + ARG efficiency   │
  ├──────────────────────┼─────────────────────────┼─────────────────────────┤
  │ Role resolution      │ Yes                     │ Yes                     │
  ├──────────────────────┼─────────────────────────┼─────────────────────────┤
  │ Principal resolution │ No ("Unknown")          │ Yes (via graphresolver) │
  ├──────────────────────┼─────────────────────────┼─────────────────────────┤
  │ Security findings    │ None                    │ Overprivilege detection │
  ├──────────────────────┼─────────────────────────┼─────────────────────────┤
  │ Exists in Aurelian?  │ No                      │ Worth building          │
  └──────────────────────┴─────────────────────────┴─────────────────────────┘
